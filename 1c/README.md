# API для синхронизации 1С и Битрикс24

Этот проект реализует REST API для синхронизации данных между 1С и Битрикс24.

## 🚀 Быстрый старт

### Установка зависимостей
```bash
npm install
```

### Настройка окружения
1. Скопируйте `.env.example` в `.env`
2. Настройте переменные окружения:
```bash
cp .env.example .env
```

Отредактируйте `.env`:
```bash
# Битрикс24
BITRIX24_WEBHOOK_URL=https://your-domain.bitrix24.ru/rest/1/your-webhook-code/
BITRIX24_USER_ID=1

# Сервер
PORT=3000

# Авторизация (ОБЯЗАТЕЛЬНО!)
AUTH_ENABLED=true
API_KEY=your-very-secure-api-key-here-123456789
```

⚠️ **Важно**: Обязательно установите собственный `API_KEY` для безопасности!

### Запуск сервера
```bash
# Продакшн
npm start

# Разработка (с автоперезагрузкой)
npm run dev
```

### Тестирование
```bash
npm test
```

## 📡 API Endpoints

### 🔐 Авторизация

Все API эндпоинты (кроме `/health`) требуют авторизации по API ключу.

**Способы передачи API ключа:**

1. **Заголовок X-API-Key (рекомендуется):**
```bash
curl -H "X-API-Key: your-api-key" http://localhost:3000/api/sync/1c
```

2. **Заголовок Authorization Bearer:**
```bash
curl -H "Authorization: Bearer your-api-key" http://localhost:3000/api/sync/1c
```

3. **Query параметр:**
```bash
curl "http://localhost:3000/api/sync/1c?api_key=your-api-key"
```

### GET `/api/sync/auth-check` 🔐
Проверка авторизации
```bash
curl -H "X-API-Key: your-api-key" http://localhost:3000/api/sync/auth-check
```

### POST `/api/sync/1c` 🔐
Основной эндпоинт для синхронизации данных из 1С

**Пример запроса:**
```bash
curl -X POST http://localhost:3000/api/sync/1c \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{"Documents": [...]}'
```

**Формат запроса:**
```json
{
  "Documents": [
    {
      "type": "Сделка",
      "GUID_Order": "guid",
      "Number_Order": "номер",
      "CompanyINN": "ИНН компании",
      "Responsible": "Ответственный",
      "products": [...]
    }
  ]
}
```

### GET `/health`
Проверка состояния сервера (не требует авторизации)
```bash
curl http://localhost:3000/health
```

## 🔄 Логика синхронизации

### Тип документа: "Сделка"
1. **Поиск сделки по GUID_Order** - если найдена, обновляется
2. **Поиск компании по ИНН** - если не найдена, создается новая
3. **Поиск ответственного по фамилии** - из поля Responsible
4. **Поиск/создание сделки** с найденной компанией и ответственным
5. **Заполнение GUID_Order** в дополнительное поле сделки
6. **Добавление товаров** из массива products
7. **Перевод в этап "Выставление счета"**

### Тип документа: "Отгрузка"
1. **Поиск сделки по GUID_Order**
2. **Смена статуса на "Отгрузка"**
3. **Сохранение текущих товаров** в дополнительное поле
4. **Замена товаров** на отгруженные из массива shipped
5. **Пересчет суммы сделки**

### Тип документа: "ЗакрытиеСделкиДосрочно"
1. **Поиск сделок по списку GUID_Order**
2. **Перевод всех найденных сделок в статус "Закрыта"**

### Создание Webhook
1. Перейдите в Битрикс24 → Приложения → Разработчикам → Другое → Входящий webhook
2. Скопируйте URL webhook и добавьте в `.env`

### Необходимые права для webhook:
- `crm` - работа с CRM
- `user` - работа с пользователями

### Пользовательские поля
В Битрикс24 нужно создать следующие пользовательские поля:

**Для сделок:**
- `UF_CRM_GUID_ORDER` (строка) - GUID заказа из 1С
- `UF_CRM_ORIGINAL_PRODUCTS` (текст) - Оригинальные товары до отгрузки

**Для компаний:**
- `UF_CRM_INN` (строка) - ИНН компании

## 📊 Этапы сделок

По умолчанию используются стандартные этапы:
- `C1:PREPARATION` - Выставление счета
- `C1:EXECUTING` - Отгрузка  
- `C1:WON` - Закрыта

Можно настроить в `controllers/syncController.js`

## 🚨 Обработка ошибок

API возвращает подробную информацию об ошибках:
- Проблемы с подключением к Битрикс24
- Неверный формат данных
- Отсутствующие поля
- Ошибки создания/обновления записей

## 🔒 Безопасность

- **API Key авторизация** - обязательная авторизация для всех эндпоинтов синхронизации
- Используется helmet для базовой защиты
- CORS настроен для кросс-доменных запросов
- Ограничение размера запроса (10MB)
